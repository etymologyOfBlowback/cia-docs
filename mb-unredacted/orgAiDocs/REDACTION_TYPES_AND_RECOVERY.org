* Overview

Analysis of redaction methods used across multiple documents revealed **two distinct approaches**:

1. **main.7: Vector Overlay Redactions** (easy recovery)
2. **main.6: Scanned Physical Redactions** (requires inpainting)

* Redaction Type 1: Vector Overlays (main.7)

** Characteristics
- Pure black vector rectangles drawn in PDF
- Located as separate PDF objects on top of image layer
- Uniform black pixels (standard RGB 0,0,0)
- Easy to detect and remove

** Recovery Method
- Extract image stream from PDF directly
- Redactions are NOT embedded in the image
- Simple extraction bypasses them entirely
- **Result**: Perfect recovery

** Files Produced
- =pdf_to_png.py=: Extracts PNG images directly
- Output: Full unredacted pages

** Test Case: iran-cia-main.7.pdf
- 22 pages total
- 14 pages with redactions
- 45 redaction boxes identified
- Recovery success: **100%** ✓

---

* Redaction Type 2: Scanned Physical Redactions (main.6)

** Characteristics
- Redactions applied BEFORE scanning
- Physical ink/paper redaction on original document
- Textured appearance in scanned image (not uniform black)
- Variation in pixel values within redacted area (std: 50-78)
- Cannot be removed with simple layer separation

** Root Cause
- Document was manually redacted on paper
- Then scanned to create the PDF
- Redaction is part of the actual image data
- **Cannot extract around it** - it's embedded in the image

** Recovery Method: Image Inpainting
Inpainting uses neighboring pixels to intelligently "fill in" redacted areas:

1. **Detect redaction mask**
   - Threshold image to find dark pixels
   - Morphological operations to refine mask
   - Result: Binary mask showing redacted regions

2. **Apply inpainting algorithm**
   - TELEA: Faster, good for small/scattered redactions
   - NS (Navier-Stokes): Slower, better for large areas
   - Uses pixel neighbors to reconstruct likely content

3. **Generate inpainted versions**
   - Two versions for comparison
   - Manual inspection needed to verify quality

** Limitations
- Inpainting cannot perfectly recover original text
- Results are plausible interpolation, not certain recovery
- Quality depends on:
  - Size of redacted area
  - Complexity of underlying text
  - Surrounding context

** Files Produced

*** Redaction Analysis
- =analyze_main6_redactions.py=
  - Detects redaction type and characteristics
  - Calculates % of page redacted per page
  - Identifies texture/variation in redacted areas

*** Inpainting Recovery
- =inpaint_main6_redactions.py=
  - Applies both TELEA and NS algorithms
  - Generates 3 files per page:
    - =page_N_inpainted_telea.png= (faster)
    - =page_N_inpainted_ns.png= (better quality)
    - =page_N_mask.png= (shows detected redactions)

*** Comparison Tool
- =create_main6_comparisons.py=
  - Creates 4-way comparison images
  - Layout: Original | TELEA
           Mask     | NS

* How to Use the Tools

** Step 1: Analyze Redaction Type

#+BEGIN_SRC bash
python3 analyze_main6_redactions.py
#+END_SRC

Output shows:
- Dark pixel percentage
- Texture variation (std dev)
- If std > 50: likely scanned redactions
- If std < 10: likely vector overlays

** Step 2: Generate Inpainted Versions

#+BEGIN_SRC bash
python3 inpaint_main6_redactions.py
#+END_SRC

Creates:
- =iran-cia-main.6-inpainted/= directory
- Two inpainted versions per page (TELEA + NS)
- Mask files for reference

** Step 3: Create Comparisons

#+BEGIN_SRC bash
python3 create_main6_comparisons.py
#+END_SRC

Creates side-by-side comparison images in:
- =iran-cia-main.6-comparison/=

** Step 4: Manual Review

Open comparison images and evaluate:
1. Does inpainted text look plausible?
2. Which method (TELEA or NS) works better?
3. Are specific redacted words visible?

* Interpreting Inpainting Results

** Good Recovery Signs
- Clear, readable text in inpainted areas
- Text aligns with surrounding context
- Consistent with document's writing style
- Multiple algorithms produce similar results

** Poor Recovery Signs
- Blurry or garbled text
- Text doesn't align with surrounding
- Different algorithms produce very different results
- Large uniform areas (can't infer content)

** Next Steps for Good Results
1. Manually compare inpainted vs original
2. Use OCR on inpainted images
3. Cross-reference with other documents
4. Apply domain knowledge (Iran 1953 context)
