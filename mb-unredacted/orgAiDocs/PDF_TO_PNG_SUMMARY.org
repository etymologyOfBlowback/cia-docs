* How It Works

** Input

A single PDF file with redacted pages:

#+BEGIN_EXAMPLE
inFile.pdf
  └─ Contains pages with black box redactions
     (redactions are vector overlays, not part of image data)
#+END_EXAMPLE

** Process

1. Extract image stream from each PDF page
2. Convert to lossless PNG format
3. Save to dedicated directory (one PNG per page)

** Output

A directory containing PNG files for each page:

#+BEGIN_EXAMPLE
inFile/
  ├─ page_1.png (unredacted - vector overlays removed)
  ├─ page_2.png
  ├─ page_3.png
  └─ ... (one PNG per page)
#+END_EXAMPLE

* Why PNG Format?

** Advantages for OCR Processing

| Feature | Benefit |
|---|---|
| Lossless compression | No quality loss (unlike JPEG) |
| Better OCR accuracy | Cleaner text edges for Tesseract |
| Standard format | Compatible with all OCR tools |
| File size impact | Minimal (acceptable for archival) |
| Text preservation | Sharp boundaries for character recognition |

** Comparison with Alternatives

| Format | File Size | Quality | OCR Speed | OCR Accuracy |
|---|---|---|---|---|
| PNG | Medium | Best (lossless) | Normal | Highest |
| JPEG | Smallest | Good (lossy) | Normal | Good |
| PDF Direct | Varies | Depends | Slow | Variable |

PNG is the optimal choice for text document OCR.

* How Redactions Are Removed

** The Mechanism

When we extract the image stream from the PDF:

1. The PDF structure contains SEPARATE LAYERS
2. Background layer: The scanned document image (unchanged)
3. Foreground layer: Vector redaction boxes (overlaid on top)
4. Image extraction bypasses the vector layer

** Why This Works

- Vector redactions are NOT destructive (don't modify the image)
- They are purely presentational (visible overlay only)
- The underlying image data remains complete and unredacted
- Simple image extraction gives direct access to the original content

** Result

The extracted PNG files are already unredacted—no additional processing is needed. The black box overlays never existed in the image data itself.

* Next Steps: OCR Processing

Once all PDFs have been converted to PNG images, the next step is to run OCR on the files to extract text.

** Basic Tesseract Usage

#+BEGIN_SRC bash
# Single file
tesseract iran-cia-main.7/page_1.png output.txt

# Batch process all pages in a directory
for png in iran-cia-main.7/*.png; do
  tesseract "$png" "${png%.png}"
done
#+END_SRC

** Batch OCR Across All PDFs

#+BEGIN_SRC bash
# Process all extracted directories
for dir in iran-cia-*/; do
  echo "Processing: $dir"
  for png in "$dir"/*.png; do
    tesseract "$png" "${png%.png}"
    echo "  ✓ Processed: $(basename $png)"
  done
done
#+END_SRC

* Troubleshooting

** Issue: "No images on this page"

Some PDF pages may not have an image layer. This is rare but can happen with:
- Text-only PDFs
- Pages with only vector graphics
- Corrupted pages

**Solution:** Check the original PDF to see if the page has visual content

** Issue: "Error converting to PNG"

The image stream may be in an unsupported format.

**Possible Causes:**
- Unusual image compression
- Corrupted image data
- Rare image format

**Solution:** Check PDF metadata or try opening with a different PDF reader

** Issue: Script not found

Make sure both scripts are in the same directory:
- =pdf_to_png.py=
- =batch_pdf_to_png.py=
