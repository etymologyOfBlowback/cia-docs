* Critical Discovery: Vector Overlay Redactions (Not Image-Based)

The fundamental finding that explains why redaction removal is so straightforward:

** The Redaction Architecture

- Each PDF page contains ONE underlying grayscale image (the actual document content, scanned)
- Black redaction boxes are NOT embedded in the image data
- Instead, they are SEPARATE VECTOR RECTANGLE objects rendered ON TOP of the image
- The PDF rendering order is: IMAGE LAYER (background) → VECTOR OBJECTS (foreground)

** Security Implication

This means:
- Extracting just the image stream from the PDF AUTOMATICALLY BYPASSES the redactions
- The redacted content is unmodified in the image data
- No sophisticated image processing or inpainting is needed
- Simple image extraction reveals all hidden content

This distinction is crucial: the redactions are visual overlays, not destructive modifications of the actual content.

* Why This Process Works: The Fundamental Vulnerability

** The PDF Architecture and Rendering Order

PDF documents are composed of multiple layers that are rendered in sequence:

1. **Background Layer**: Image objects (the scanned document content)
2. **Content Layers**: Vector graphics, text, shapes
3. **Foreground Layer**: Annotations, overlays, redaction boxes

The rendering process is: IMAGE → VECTOR OBJECTS → REDACTION OVERLAYS

When a PDF viewer renders the page, it draws these layers in order, with later layers appearing on top of earlier ones. This means the black redaction rectangles are drawn LAST, on top of everything else.

** Why Image Extraction Bypasses Redactions

When we extract the image stream from the PDF using =pdfplumber=:

#+BEGIN_SRC python
stream = page.images[0].get('stream')
raw_data = stream.get_data()
#+END_SRC

We are accessing the PDF's internal image object directly, completely bypassing the vector rendering layer. This is like looking at the canvas before the redaction layer was painted on top—the image data is untouched and complete.

The vector redaction boxes are stored separately from the image and exist only in the PDF's graphical state objects. By extracting the image independently, we completely circumvent them.

** The CIA Documents: A Case Study in Poor Redaction Practice

The iran-cia-*.pdf documents exhibit multiple security failures:

| Issue | Impact |
|---|---|
| Vector-only redactions | Easily bypassed by image extraction |
| No image modification | Original content remains intact |
| Separate overlay layer | Can be removed without touching image |
| No encryption | Anyone can access PDF internals |
| No verification | No way to detect extraction attempts |

This represents a fundamental misunderstanding of PDF security. The creators believed that visual obscuring would prevent information access, not realizing that PDFs maintain internal structure that can be accessed programmatically.

** A Proper Redaction Would Require

For truly secure redaction, the document creators would have needed to:

1. **Modify the image data directly** - Turn pixels black/transparent in the actual JPEG/image data
2. **Use encryption** - Encrypt sensitive portions of the PDF
3. **Flatten the document** - Merge all layers irreversibly
4. **Verify integrity** - Use checksums to detect tampering
5. **Audit access** - Track who has accessed the document

None of these were done with the iran-cia documents.

* Limitations and Edge Cases

** When This Process May Not Work

1. **When redactions are genuinely embedded**: If the redactor actually removed/blanked pixels from the image before creating the PDF
2. **Multiple overlapping redactions**: Can make boundary detection difficult
3. **Complex visual content**: Text/diagrams with extreme variations may not interpolate well
4. **Corrupted overlays**: If the black box contains transparency gradients or patterns

* Automation: Bulk Processing of Iran CIA Documents

The simplicity of the image extraction method makes it trivial to process all 17 iran-cia-*.pdf files in bulk.

** Batch Processing Script

#+BEGIN_SRC python
#!/usr/bin/env python3
"""
Extract all images from iran-cia-*.pdf files, automatically removing redactions.
"""

import pdfplumber
from PIL import Image
from pathlib import Path

pdf_dir = Path('/bxo/iso/pip_archive-cvs2014/doc/paykNet/politics/1953MossadeqCoup/cia-53-iranCoup/pdf-files-orig')
output_dir = pdf_dir / 'unredacted_pages'
output_dir.mkdir(exist_ok=True)

pdf_files = sorted(pdf_dir.glob('iran-cia-*.pdf'))

print(f"Processing {len(pdf_files)} PDF files...\n")

for pdf_path in pdf_files:
    print(f"Processing: {pdf_path.name}")
    
    with pdfplumber.open(pdf_path) as pdf:
        for page_num, page in enumerate(pdf.pages, 1):
            if page.images:
                img_info = page.images[0]
                stream = img_info.get('stream')
                
                # Extract raw image data
                raw_data = stream.get_data()
                
                # Determine output filename
                base_name = pdf_path.stem
                output_filename = f"{base_name}_page_{page_num}.jpg"
                output_path = output_dir / output_filename
                
                # Save JPEG
                with open(output_path, 'wb') as f:
                    f.write(raw_data)
                
                # Also save as PNG for easier viewing
                try:
                    img = Image.open(output_path)
                    png_path = output_dir / f"{base_name}_page_{page_num}.png"
                    img.save(png_path)
                except:
                    pass
                
                print(f"  ✓ Page {page_num}: Extracted")

print(f"\n✓ Complete! All unredacted pages saved to: {output_dir}")
#+END_SRC

- Each page extracted as individual JPEG and PNG files
- All redactions automatically removed (they're not part of the image stream)

* Security Implications

This process demonstrates an important security principle:

** Redaction ≠ Deletion**

- Visual redaction (overlays) does not destroy information
- It only hides information from casual viewers
- Anyone with access to the raw data can potentially recover it
- Proper information security requires actual data destruction, not covering

** Secure Redaction Requires:**

1. Permanent removal of sensitive data from the image stream
2. Encryption of sensitive regions
3. Multiple rendering passes to ensure complete overlay
4. Verification that underlying data is genuinely removed
